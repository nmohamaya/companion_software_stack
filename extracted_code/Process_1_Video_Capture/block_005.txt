Context: V4L2 (Video4Linux2) is the Linux kernel API for video capture devices. The capture flow is:
Source: table_row0_col0
---
    Application                          Kernel / Driver
    ───────────                          ──────────────
    1. open("/dev/video0")  ──────────►  Bind to camera driver
    2. VIDIOC_QUERYCAP      ──────────►  Verify V4L2_CAP_VIDEO_CAPTURE
    3. VIDIOC_S_FMT         ──────────►  Set resolution, pixel format
    4. VIDIOC_REQBUFS (MMAP)──────────►  Allocate N kernel DMA buffers
    5. VIDIOC_QUERYBUF × N  ──────────►  Get buffer offsets
    6. mmap() × N           ──────────►  Map DMA buffers to userspace
    7. VIDIOC_QBUF × N      ──────────►  Enqueue all buffers to driver
    8. VIDIOC_STREAMON       ──────────►  Start capture
    
    ┌─── CAPTURE LOOP (30 fps) ───────────────────────────────────────┐
    │                                                                   │
    │  9. poll(fd, POLLIN)    ◄─────────  Wait for frame interrupt     │
    │  10. VIDIOC_DQBUF      ◄─────────  Dequeue filled buffer        │
    │  11. process(buffer)               Push to SPSC queue            │
    │  12. VIDIOC_QBUF       ──────────►  Re-enqueue buffer           │
    │                                                                   │
    └─────────────────────────────────────────────────────────────────┘
    
    13. VIDIOC_STREAMOFF      ──────────►  Stop capture (on shutdown)
